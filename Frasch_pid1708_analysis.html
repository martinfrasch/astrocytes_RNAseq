1 Introduction

Please note that this is a re-analysis of these data, where we exclude animal 4786 from the microglia samples, and use conventional linear models rather than generalized linear models, in order to fit array weights to the microglia samples to down-weight the one sample with exceedingly low counts. We keep the generalized linear model for the astrocyte analysis.

This is an analysis of fetal sheep astrocytes that were controls or exposed to LPS, LPS plus an agonist or LPS plus an antagonist. One of the sheep was exposed to LPS in vivo as well as in vitro. We characterize this animal as a ‘double hit’ and the others as ‘single hit’. The table below describes the set of samples for this experiment.

The goal for this analysis is to compare the single hit LPS treated samples to Control, as well as making comparisons between the LPS treated samples (e.g., testing for changes due to the additional agonist or antagonist treatment). The double hit samples have no replicates, so we can only make the directed comparisons between the single and double hit samples. We aligned the reads to the Oar_v3.1 transcriptome using the salmon aligner (Patro, Duggal, and Kingsford 2015), which infers the most likely transcript for each read using a quasi-mapping algorithm. We then ‘collapse’ the transcript read counts to the gene level by summing up the reads for each gene’s transcripts, using the Bioconductor tximport package (Soneson, Love, and Robinson 2015). In the end we have a set of read counts per gene, for each sample.

PCA, or principal components analysis, is a method to take high dimensional data and reduce to only a few dimensions in order to visualize how similar samples of the same type are. The first dimension captures the largest amount of variability between samples, the second dimension captures the next largest amount of variability, and so on. 3 Model fit and results

To make these comparisons we use the Bioconductor limma package (Smyth 2004), which includes a function that is intended to convert the count data to log counts/million, as well as estimating observation-level weights (e.g. a weight for each gene, for each sample). We can then fit a conventional weighted linear model to these data, as the weights account for any dependence between the mean and variance. For the microglia samples we also compute sample-specific weights that are intended to down-weight any samples that are not very similar to other samples of the same type. This is an alternative to simply excluding the sample.

We fit the model described above, including a blocking effect for each animal, and made the requested comparisons, incorporating a fold change criterion into the test. In other words, the conventional test for a difference is between the null hypothesis \(H_0: \beta = 0\) versus the alternative hypothesis \(H_A: \beta \neq 0\), but this may include very small changes that are likely to be biologically meaningless. One alternative to exclude such genes is to use a post hoc fold change adjustment, where we select genes based on the observed fold change between groups. This is problematic because we ignore the imprecision in our estimate of fold change. A better method is to incorporate the fold change into our inference, where we test \(H_0: \beta < |c|\) against the alternative \(H_A: \beta \geq |c|\), for some constant fold change. By doing this, we are testing to see if we have evidence that the underlying population differences are larger than a given fold change, rather than simply testing that our sample data fulfill those criteria. We used a 1.5-fold change, and a false discovery rate (FDR) of 0.05 as criteria to define significantly differentially expressed genes.

The links in the above table will load HTML tables containing the genes that were significantly different in this analysis. By default the table shows only 10 rows, and is sorted on the first column. You can change the number of rows using the drop-down at the top left of the page, and can sort on any column by simply clicking on the header of the column you want to sort by. Each row has a small plot showing the relative differences between the two groups, but in order to fit the plot into the table we have to strip off the axis labels. If you right-click on any plot and choose ‘Open in new tab’, you will be presented with a full size plot that does have axis labels.

The first four columns in the table contain annotation information, including links to Ensembl and NCBI, and the gene symbol and description (if available). We get the symbol and description from Ensembl’s Biomart server, and for some reason not all of the symbols and/or descriptions are available from that database. The logFC column shows the log fold change between the groups being compared. The sign of the fold change indicates if the gene is up or down-regulated, and the title of the table indicates the direction of comparison. For example, LPS100_1_hit vs Ctrl_1_hit indicates that we are comparing the LPS treated - Control for the single hit samples, so a positive fold change means the gene is up-regulated in LPS treated samples, and a negative fold change indicates it is down-regulated in LPS treated samples. You can convert the log fold changes to linear scale by exponentiating; if the logFC is 1.23, then the linear fold change is \(2^{1.23}\) = 2.35.

The logCPM column can be interpreted as a measure of the relative expression for a given gene. All things equal, a larger value indicates a higher level of gene expression. The FDR is a is a measure that estimates the maximum proportion of false discoveries in a given set of genes. We selected genes at an FDR < 0.05, meaning that we expect at most that 5% of the genes selected are false positives.

We also output the same set of tables in Excel format - you can find them in the spreadsheets subdirectory that accompanies this document. In addition, we output all (post filtered) genes in two Excel spreadsheets that is primarily intended for analysis using IPA (Ingenuity Pathway Analysis), called Frasch_pid1708_astroglia_ingenuity.xlsx and Frasch_pid1708_microglia_ingenuity.xlsx. These files can be useful to look at results for a given gene, across all comparisons, or to simply look for genes that might not appear in the sets of differentially expressed genes.

3.1 Venn diagrams

There are several sets of comparisons with consistent denominators, such as the three comparisons of treated samples to control samples. It might be of interest to know which genes are unique to a given comparison, and which are shared between two or more of those comparisons. We generated Venn diagrams for three sets of comparisons.

We made a Venn diagram for the three comparisons of treated vs control, a two-way Venn diagram of the agonist + LPS versus either LPS alone or LPS plus antagonist (this shows that there is very little difference between LPS and LPS plus antagonist), and finally we make a Venn diagram of the one-hit versus two-hit for all three treatments.

Please note that the genes in any intersection between comparisons have to be significantly differentially expressed in both (or all three, depending on the intersection) of the comparisons. In addition, the direction of change has to be the same as well. For example, if a given gene is differentially expressed in LPS vs control and LPS+A10 vs control, and it is either up or down-regulated in both comparisons as well, then it will be listed in the intersection between those two comparisons. If it is significant in both comparisons, but is up-regulated in one comparison, but down-regulated in the other, then it will be listed in the unique portion of the Venn diagram for each comparison.

The numbers in each cell of the Venn diagrams are links that will open HTML pages containing the genes in a given cell. The ‘venns’ subdirectory that accompanies this report also contains text versions of the data in each cell of each Venn diagram.

3.2 Comparison to microglia samples

Please note that differences between the astrocyte and microglia samples are confounded by technical differences. In other words, we cannot say if any differences between the two cell types represent real biological differences or are simply technical differences due to running samples at different times, or some unknown combination of the two.

One additional goal for this analysis was to compare to the previous microglia samples. To do so, we downloaded the FASTQ data from SRA and processed using the same salmon/tximport pipeline we used for the astrocyte samples. The only difference was in the modeling step, where we used the GLM model instead of the quasi-likelihood model. The quasi-likelihood model is more conservative than the GLM (and arguably suffers from fewer false positives). With the microglia data we get no differentially expressed genes when using the quasi-likelihood, so we had to use the GLM framework. We still incorporated the 1.5-fold change into the comparisons and selected genes based on an FDR < 0.05, so the only difference is how we model the gene dispersion.

First, we can take a look at a PCA plot that includes just the single-hit LPS and control samples for microglia and astrocytes (we exclude the two-hit samples because there is no two-hit control for astrocytes).

From the above plot it is obvious that the main differences between samples are captured on the first two principal components, so a 2D plot may be more useful. Regardless, a high resolution tiff can be found here. One other observation we might make is that the intra-group variability is smaller for the astrocytes as compared to the microglia. This may have to do with the total library sizes - recall that there are several microglia samples with very few reads that aligned to any known transcript.

A high resolution tiff of the 2D plot can be found here.

We can do a few different things to compare the astrocyte and microglia results. Given the confounding between the two experiments, it is not advisable to make any direct comparisons between microglia and astrocyte samples. But we can still make comparisons.

The simplest and most naive thing to do would be to compare the genes that are differentially expressed in similar comparisons. For example, the set of genes that are different between LPS and control for the two different cell types. We can use Venn diagrams, as above, for this task. Note that, as with the previous Venn diagrams, we require that the genes in any intersection between two or more comparisons have to be significant in both comparisons, as well as having the same direction of change.

Alternatives to Venn diagrams would be some sort of gene set testing, where we try to detect pathways that are affected in both cell types. This sort of analysis is far more complex, and if you want to do something like that we should have a meeting to discuss the parameters first.
